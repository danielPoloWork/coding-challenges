# Notes - LeetCode 3509: Maximum Product of Subsequences With an Alternating Sum Equal to K (C proposal)

## Problem Summary

Choose a non-empty subsequence whose alternating sum is exactly `k` and whose product
does not exceed `limit`. Return the maximum such product, or `-1` when no subsequence
qualifies.

## Proposal in This Folder (C)

This folder holds the **C** minimum-memory proposal:

- **Memory extreme (`solution-memory.c`):** the same compressed product-state DP as the
  C++ champion, but implemented with exact-width manual `uint64_t` bitsets sized to
  `2 * sum(nums) + 1` instead of a fixed maximum-width C++ bitset.

The recommended and fastest-runtime proposal is implemented in C++ - see
[See Also](#see-also).

## Language Choice (C)

C minimizes runtime-library overhead and gives direct control over the exact number of
64-bit words allocated for each bitset row. That is the best fit for the least-memory
objective, especially when `sum(nums)` is much smaller than the maximum 1800.

## Constraints and Observations

- `sum(nums) <= 1800`, so any `k` outside that range is impossible.
- Positive products are bounded by `limit <= 5000`.
- Products come only from factors `1..12`; even with all factors present, there are at
  most 394 distinct positive products under 5000.
- A product-0 answer can come from a subsequence whose positive prefix would have
  exceeded `limit`, so zero reachability must not depend on the bounded positive-product
  DP.

## Approach

1. Build the compressed sorted list of positive products generated by values present in
   `nums`.
2. Allocate two exact-width bitset rows per product: odd subsequence length and even
   subsequence length.
3. Maintain four extra exact-width bitsets:
   - all non-empty odd/even subsequences;
   - zero-containing odd/even subsequences.
4. Append each `x` by shifting sum bitsets left for `+x` and right for `-x`.
5. Process product rows descending for `x > 1`; copy the current row before toggling for
   `x == 1`.
6. Scan products descending for the best positive result, then fall back to product `0`
   if a zero-containing subsequence reaches `k`.

## Trade-Off

The C variant reduces memory by using exact dynamic bit widths and avoiding C++ container
overhead. The C++ solution remains preferable for the default and runtime objective
because `std::bitset` produces compact, highly optimized word shifts with simpler code.

## Top 1% Performance Strategy

Even the memory variant keeps the same performance architecture: compressed product
rows, word-level shifts, early impossible-`k` rejection, and separate zero handling.
